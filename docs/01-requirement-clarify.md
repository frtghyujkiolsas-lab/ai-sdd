# 01 需求澄清

> 本文只记录 PRD 前的澄清结论：用户是谁、输入输出是什么、边界在哪里。完整功能范围与验收标准见 [02 PRD](02-prd.md)。

## 原始需求（一句话）

员工上传任意生活照，系统产出统一效果的公司证件照，并合成标准工牌。

## 需求澄清问题

| 问题 | 当前结论 |
| --- | --- |
| 谁使用这个系统？ | HR、行政或入职办理人员 |
| 输入是什么？ | 员工生活照、姓名、部门、岗位、工号 |
| 输出是什么？ | 标准证件照和企业工牌 PNG |
| 什么叫“统一效果”？ | 头像比例、背景、工牌版式、字段位置统一 |
| 哪些内容必须可改？ | 姓名、部门、岗位、工号、有效期等字段 |
| 一期不做什么？ | 复杂权限、批量审批、多模板商城、正式员工系统集成 |

## PRD 前置结论

- 业务目标：降低 HR / 行政在头像收集、审核、套模板上的重复劳动。
- 产品边界：一期先验证单人单张生成闭环，不扩展成完整 HR 管理后台。
- AI 边界：AI 只负责头像标准化，不负责自动判断员工字段真实性。
- 交付边界：最终输出可下载 PNG，便于展示、转发或打印前预览。

## 核心约束

- 照片来源不可控，需要 AI 标准化。
- 员工字段不能完全交给 AI，必须可人工确认。
- 最终结果需要是可直接保存和转发的图片。
- 公开仓库不提交真实员工照片和真实工牌数据。
