# AI-SDD 流程总览

本仓库用 **AI 智能工牌生成系统** 演示一套轻量的 AI 产品交付流程。目标不是写厚文档，而是让需求、原型、接口、实现和验收之间可以互相追踪。

## 标准流程

```text
需求澄清
→ PRD 定义
→ 原型定稿
→ 接口契约
→ 后端逻辑
→ 前后端联调
→ 交付验收
```

## 样例需求

> 员工上传任意生活照，系统自动生成统一风格证件照，并合成公司的标准工牌，最终导出 PNG。

## 每一步的产物

| 阶段 | 解决的问题 | 关键产物 |
| --- | --- | --- |
| 需求澄清 | 业务到底要什么，边界在哪里 | 目标、用户、场景、约束 |
| PRD 定义 | 哪些功能必须做，哪些先不做 | 功能范围、流程、验收标准 |
| 原型定稿 | 页面和交互是否达成一致 | 页面流、状态、字段 |
| 接口契约 | 前后端按什么数据结构协作 | API、字段、错误码 |
| 后端逻辑 | AI 处理和模板合成怎么跑 | 处理链路、依赖、兜底 |
| 前后端联调 | Mock 到真实接口是否一致 | 联调记录、异常场景 |
| 交付验收 | 最终结果是否可用 | 测试清单、交付截图 |

## 为什么要这样组织

- 避免需求只停留在聊天记录里。
- 避免前端 Mock、后端接口和最终效果不一致。
- 让面试官或协作者能快速看懂：从模糊需求到可运行 Demo，中间经历了哪些产品判断。
