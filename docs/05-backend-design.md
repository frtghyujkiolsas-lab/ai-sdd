# 05 后端逻辑

## 处理链路

```text
接收表单数据
→ 校验照片和员工字段
→ 调用 HivisionIDPhotos 生成证件照
→ 注入 HTML/CSS 工牌模板
→ 浏览器渲染模板
→ 截图导出 PNG
→ 返回 base64 结果
```

## 关键设计

### 1. 图像标准化

通过 HivisionIDPhotos 处理头像检测、裁剪、背景替换和证件照规格化。这样可以把“任意生活照”转换成较统一的头像输入。

### 2. 模板合成

工牌不是直接由大模型生成，而是使用 HTML/CSS 模板合成。这样姓名、岗位、部门、工号等字段位置可控，输出风格稳定。

### 3. PNG 导出

后端渲染最终 HTML 后截图，输出 PNG。这样前端只负责展示结果，不需要重复实现工牌渲染逻辑。

## 异常处理

| 异常 | 处理方式 |
| --- | --- |
| 未上传照片 | 返回明确错误提示 |
| 证件照服务不可用 | 提示检查 HivisionIDPhotos 服务 |
| 图片处理失败 | 提示更换清晰正脸照片 |
| 模板渲染失败 | 返回生成失败并保留日志 |
