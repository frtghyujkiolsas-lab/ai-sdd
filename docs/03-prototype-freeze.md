# 03 原型定稿

## 页面清单

| 页面 | 作用 |
| --- | --- |
| 上传页 | 收集照片和员工字段 |
| 处理中页面 | 展示 AI 处理进度和任务状态 |
| 结果页 | 展示标准证件照和最终工牌 |
| 管理页 | 展示 HR 视角的生成记录 |

## 定稿原则

- 页面数量少，优先保证核心闭环。
- 上传页字段与后端接口字段保持一致。
- 结果页不重新绘制工牌，而是展示后端生成的最终 PNG。
- 管理页作为展示用，不阻塞主流程。

## 原型状态

当前前端原型位于：

```text
figma-make-export/
```

截图位于：

```text
screenshots/
```

## 冻结后的变更规则

原型定稿后，如果要调整字段、页面状态或数据结构，需要同步更新：

- PRD 功能范围
- API 契约
- 前端表单字段
- 后端生成逻辑
- 联调测试用例
